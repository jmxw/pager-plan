# 11002 DeployTaskSkipDueToStarted

> Category: Deploy Task | HTTP Status: N/A — raised only inside the `app/consumer` deployment worker (RabbitMQ context, not an HTTP request). It is logged and produces no HTTP response.

A deploy task was skipped because it had already started.

## Error Message

`Deploy task({}) skipped, because it was started, current state is {}, cannot be ran again.`

## Placeholders

| Placeholder | Description |
| --- | --- |
| `{}` (1st) | The deploy task id. |
| `{}` (2nd) | The current `DeployState`. |

## When Returned

Thrown by [`ProjectDeployService`](/source/pager-backend/app/consumer/src/main/kotlin/com/xwkj/pager/consumer/service/ProjectDeployService.kt) in the consumer when the [deploy task](/doc/tech_noun/TN0701_deploy_task.md) state is not `SUBMIT` (it was already started and cannot run again).
