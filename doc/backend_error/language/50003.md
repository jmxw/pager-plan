# 50003 LanguageDefaultNotFound

> Category: Language | HTTP Status: `500 Internal Server Error` — raised as a `PagerInternalException`. After it is logged at its enum log level, [`CmsExceptionHandler`](/source/pager-backend/app/cms/src/main/kotlin/com/xwkj/cms/exception/CmsExceptionHandler.kt) answers the request with [`-9999 UnknownError`](/doc/backend_error/unknown/-9999.md), so this code never reaches the client.

A project has no language flagged as its default.

## Error Message

`Default language not found in project({}).`

## Placeholders

| Placeholder | Description |
| --- | --- |
| `{}` | The project id. |

## When Returned

Thrown by [`LanguageRepository`](/source/pager-backend/domain/src/main/kotlin/com/xwkj/pager/domain/repository/LanguageRepository.kt) (`getDefaultByProject`) when a [project](/doc/tech_noun/TN0301_project.md) has no [language](/doc/tech_noun/TN0302_language.md) flagged `defaulted`. Reached while adding a list or article [navigation](/doc/tech_noun/TN0601_navigation.md) (which resolve the default language). It is a `PagerInternalException` representing a data-integrity fault — every project is expected to keep exactly one default language.
