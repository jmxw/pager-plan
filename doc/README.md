# Reference Docs

Cross-cutting reference material for Pager, produced by following the authoring conventions in
[`../rules/`](../rules/README.md). Plans link into these docs with **root-absolute** paths (`/doc/...`).

- [`api/`](api/README.md) — one doc per API endpoint, grouped by controller.
- [`backend_error/`](backend_error/README.md) — one doc per backend error code, grouped by category.
- [`permission/`](permission/README.md) — one doc per `UserPermission` value.
- [`security/`](security/README.md) — security audit findings (backend / frontend).
- [`spec/`](spec/README.md) — spec index, the master catalog of Pager CMS screens.
- [`tech_noun/`](tech_noun/README.md) — domain-term definitions, the source of terminology for all docs.
